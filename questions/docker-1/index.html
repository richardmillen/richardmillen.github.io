Docker Quiz

Quiz 1

  1. How do you run a container with interactive shell access?
  2. How do you commit changes to a new Docker image instance?
  3. How can you view the container id from an interactive shell?
  4. How do you list the docker images on your machine?
  5. How do you view active containers?

1.

docker run -it image-name

2.

docker commit -m "<commit message...>" -a "<author...>" container-id repo-name/new-image-name

3.

The container prompt is in the format root@container-id.

4.

docker images

5.

docker ps

Quiz 2

  1. How do you view all containers, active and inactive?
  2. How do you stop an active container?
  3. How do you connect to docker hub?
  4. How do you publish your own image to a registry?
  5. What is detached mode?

1.

docker ps -a

2.

docker stop container-id

3.

docker login -u <registry-username>

4.

docker push <registry-username>/<image-name>

5.

Container is started and stays running until stopped but does not listen to the command line.

Quiz 3

  1. What does docker run -P image-name do?
  2. What does docker run -d <image-name> do?
  3. How do you create a container instance in detached mode, publishing all exposed ports to random ports and also give it a name?
  4. What does docker run -p do?
  5. How do you mount one or more host directories to directories in a container?

1.

Runs a container, publishing all exposed ports to random ports between 49153 and 65535.

2.

Runs a container in detached mode.

3.

docker run --name <container-name> -d -P <image-name>

4.

Runs a container, published all exposed ports to the host.

5.

docker run -v host-path:container-path -v host-path:...

Quiz 4

  1. How do you mount a host directory to read-only directories in a container?
  2. What command do you run to create an image from a docker file?
  3. How can you edit state in a detached container, even though SSH access is not available?
  4. What does docker run -i <image-name> do?
  5. What does docker run -t <image-name> do?

1.

docker run -v host-path:container-path:ro

2.

With a Dockerfile in the working directory:

docker build -t image-name .

3.

Use a helper container that has shell access.

e.g.

docker run --name target -d image-name
docker run -it --volumes-from target --name helper image-name /bin/bash

4.

Runs a container with STDIN open.

5.

Runs a container with a (pseudo) TTY / shell.

Quiz 5

  1. What operating system is used by the NGINX image on Docker Hub?
  2. How do you exit a container while leaving it running?
  3. How do you activate an existing container?
  4. How do you gain shell access to an active container?
  5. What happens to error logs written to the Docker log collector?

1.

Debian

2.

CTRL+P, CTRL+Q

3.

docker start container-name

4.

docker attach container-name

5.

Stored in the file /var/lib/docker/containers/<long-id>/<long-id>-json.log on the Docker host.

Quiz 6

  1. How do you view the long form container id?
  2. How can you extract log messages of a container written by the Docker log collector?
  3. How do you restart a container?
  4. How do you get the ip address of a container from the host?
  5. …

1.

docker inspect --format '{{ .Id }}' container-name

2.

docker logs container-name

3.

docker restart container-name

4.

docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' container-name-or-id

5.

docker run --name container-name --restart always -d -v /path/:/path/ -v /path/file:/path/file --add-host=name:addr image-name:latest
