  3. What is the ~ operator?
  4. What is the << operator?
  5. What is the >> operator?

1.

Binary.
Evaluated left-to-right.
Result is the value of the right-hand operand.
Result is lvalue if operand is lvalue.

2.

Bitwise exclusive or.
Generates new integral value in which each position is 1 if either but not both operands contain a 1, otherwise 0.

3.

Bitwise NOT operator.
Generates new integral value in which each bit is an inverted copy of those in the (possibly promoted) operand.

4.

Left-shift operator.
Left-hand operand should be unsigned.
If the left-hand operand is signed, it is undefined if a shift causes a different bit to shift into the sign bit.

5.

Right-shift operator.
Opposite to left-shift operator.
If the left-hand operand is signed, it is implementation defined whether bits shifted into the result are 0 or a copy of the sign bit.

When the copy constructor is executed, it (recursively) calls the copy constructor due to the pass-by-value parameter.
Endless recursive calls cause a stack overflow.

CMyString& CMyString::operator=(const CMyString& str) {
    if (this != &str) {
         CMyString strTemp(str);
         char* pTemp = strTemp.m_pData;
         strTemp.m_pData = m_pData;
         m_pData = pTemp;
    }
    return *this;
}

Key points to look out for:-

  • Does it return a reference? Without it, it’s not possible to chain assignments.
  • Is the argument passed by a constant reference? If it’s by value then a copy is made which is wasteful.
  • Is the existing memory freed?
  • Does it protect against self-assignment?
  • What if an exception is throw while reallocating memory? This would leave the object in an invalid state.

Quiz 20

  1. How does a deque differ from a vector?
  2. How do you define a type that may be used as though it were a function?
  3. When are function objects typically used?
  4. What is the thread class?

1.

Similar to vectors, but with efficient insertion and deletion at the beginning, not only at its end.
Vectors use a single array that needs to be reallocated occasionally for growth.
Elements of a deque can be scattered in different chunks of storage.
More complex internally than vectors.
Grow more efficiently under certain circumstances, especially with very long sequences, where reallocations become more expensive.

2.

By defining member function operator() in the class.
e.g.

struct myclass {
    int operator()(int a) {return a;}
} myobject;
//...
int x = myobject(0);

3.

As arguments to functions, such as predicates or comparison functions passed to standard algorithms.

4.

An initialised thread object represents an active thread of execution.
An initialised thread object is joinable.
An initialised thread object has a unique thread id.
A default-constructed (non-initialised) thread object is not joinable and has a common thread id.
A joinable thread becomes not joinable if move-assigned, or join() or detach() is called.
Thread objects cannot be copied.
Calling the join() method on a thread causes the calling code to pause until the thread finishes.

5.

Prevents concurrent access to code.
It’s a mutex wrapper that provides a convenient RAII-style mechanism for owning a mutex for the duration of a scoped block.
e.g.

#include <mutex>
void doItSafely() {
    std::lock_guard<std::mutex> block(mymutex);
    // from this point until the 'block' variable goes out of scope,
    // we're thread safe!
}
