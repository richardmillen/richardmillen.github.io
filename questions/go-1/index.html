Quiz 19

  1. What helps to make the Go compiler fast?
  2. Why might it be practical to use Go where a scripting language might normally be used?
  3. Explain why many Go programs using concurrency don’t need to do any locking?
  4. What are goroutines, how do they work and how are they called?
  5. What are channels?

1.

  • The language is easy to parse.
  • Go does not use a preprocessor or include files.
  • The compiler only needs to look at the libraries that you directly include, rather than traversing the dependencies of all the libraries that are included in the entire dependency chain like Java, C, and C++. e.g. everything that pkg2.go exports is cached in pkg1.go’s object file, so pkg1.go’s object file alone is sufficient to build app.go.

2.

Because Go programs are so fast to build.

3.

Go provides a form of CSP (Communicating Sequential Processes) based on the ideas of computer scientist C. A. R. Hoare.

4.

Functions that run concurrently with other goroutines, including the entry point of your program.
They are like threads, but use far less memory and require less code to use.
They can be created in vast numbers that are automatically load-balanced across the available processors and cores.
The Go runtime schedules their execution against a set of configured logical processors.

go foo(arg1, arg2)

5.

They are data structures that let you send typed messages between goroutines with synchronization built in.
Help to enforce the pattern that only one goroutine should modify the data at any time.
They don’t provide data access protection between goroutines.
If copies of data are exchanged any changes can be made safely. If pointers are exchanged read / write synchronisation may be needed.
They implement a queue of typed values that are used to communicate data between goroutines.
