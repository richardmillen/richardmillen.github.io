Quiz 6

  3. What are some of the benefits of using Linux containers over VMs?
  4. What are some downsides of Linux containers?
  5. How do you copy the contents of a text file to the clipboard from a bash shell?

1.

Each container is effectively a subtree of the overall system process tree. These containers can have physical resources allocated to them, something the kernel handles for us. This general approach has been around in many forms, such as Solaris Zones and OpenVZ, but it is LXC that has become most popular. LXC is now available out of the box in any modern Linux kernel.

2.

”> host running LXC > we donâ€™t need a hypervisor. > although each container can run its own operating system distribution, it has to share the same kernel ~ because the kernel is where the process tree lives. > our host operating system could run Ubuntu, and our containers CentOS, as long as they could both share the same kernel.”

3.

”> Linux containers are much faster to provision than full-fat virtual machines. > Resources saved by not needing a hypervisor. > Finer-grained control over the containers themselves in terms of assigning resources to them. > Due to the lighter-weight nature of containers, we can have many more of them running on the same hardware than would be possible with VMs.”

4.

”> The outside world can’t see them and configuring port forwarding can be timeconsuming. > These containers cannot be considered completely sealed from each other. ~ There are many documented and known ways in which a process from one container can bust out and interact with other containers or the underlying host.”
