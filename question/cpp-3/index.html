2

How many associative containers are provided in the standard library?


8 - Each is either a map or a set:

  1. set
  2. multiset
  3. map
  4. multimap
  5. unordered_set
  6. unordered_multiset
  7. unordered_map
  8. unordered_multimap

3

What is an associative array?


Elements indexed by key rather than positionally.

4

What is an associative container?


Collection that supports efficient lookup by key.

5

What is a hash?


Library template.
Used by unordered containers to manage position of their elements.

6

What is a key_type?


Type used and defined by associative containers.

7

What is a map?


Associative container.
Class template.
Each element must have a unique key.
Dereferencing an iterator yields a pair.

8

What is a mapped_type?


Type defined by map types.
Type of the values associated with the keys in a map.

9

What is a multimap?


Associative container.
Similar to map, except a given key may appear more than once.
Doesn’t support subscripting.

10

What is a multiset?


Associative container.
Similar to set, except a given key may appear more than once.

11

What is a pair?


Type with two public data members:

  1. first
  2. second

12

What is a set?


Associative container.
Holds keys.
A given key may appear only once.
key_type and value_type are the same.

13

What is strict weak ordering?


Relationship among keys in associative container.
Possible to compare any two and determine which is ‘less’, if not they’re considered equal.

14

What is an unordered container?


Associative container.
Uses hashing rather than comparison operation on keys to store / access elements.
Performance depends on hash function.

15

What is an unordered_map?


Associative container.
Elements are key-value pairs.
Permits only one element per key.

16

What is an unordered_multimap?


Associative container.
Elements are key-value pairs.
Allows multiple elements per key.

17

What is an unordered_multiset?


Associative container.
Stores keys.
Allows multiple elements per key.

18

What is an unordered_set?


Associative container.
Stores keys.
Permits only one element per key.

19

What is an allocator?


Library class.
Defined in <memory>.
Lets you separate allocation from construction.
Sometimes you allocate a block of memory you plan to use to construct objects as needed. This way you pay the overhead of constructing objects only when you actually need to create them.
Provides type-aware allocation of raw, ‘unconstructed’ memory.
Allocates memory that is appropriately sized and aligned to hold the objects.
User must construct objects in order to use memory returned by allocate.
User must call destroy on objects that were constructed in this memory before calling deallocate.
e.g.

allocator<string> alloc;
auto const p = alloc.allocate(n);
auto q = p;
alloc.construct(q++);
alloc.construct(q++, 10, 'c');
alloc.construct(q++, "hi!");
//...
while (q != p)
    alloc.destroy(--q);
alloc.dealloc(p, n);"

20

Which companion functions are there to the allocator class?


  1. uninitialized_copy - Copies elements denoted by ‘begin’ and ‘end’ iterators, into unconstructed, raw memory denoted by a third iterator.
  2. uninitialized_copy_n - Copies ‘n’ elements starting from ‘begin’ iterator, into unconstructed, raw memory denoted by a second iterator.
  3. uninitialized_fill - Constructs objects into raw memory denoted by ‘begin’ and ‘end’ iterators, as a copy of a third parameter.
  4. uninitialized_fill_n - Constructs ‘n’ objects starting from ‘begin’ iterator, which denotes unconstructed, raw memory large enough to hold the given number of objects.

These functions construct elements in the destination, rather than assigning to them.

21

Using the two companion algorithms of the allocator class, implement the following:

  1. allocate twice as many elements as variable ‘vi’ holds,
  2. construct elements (starting at the beginning of the allocated memory), as copies of elements in vi.
  3. initialise the remaining elements to 42.

auto p = alloc.allocate(vi.size() * 2);
auto q = unintialized_copy(vi.begin(), vi.end(), p);
uninitialized_fill_n(q, vi.size(), 42);

22

What is a dangling pointer?


Pointer that refers to memory that once had an object but no longer does.
Difficult to debug.

23

What is a deleter?


Function passed to a smart pointer.
Used in place of ‘delete’.

24

What does dynamically allocated mean?


Object is allocated on the free store.
They exist until explicitly deleted or program terminates.

25

What is the free store?


Memory pool available to a program to hold dynamically allocated objects.

26

What is placement new?


Form of new that takes additional arguments.
e.g.

new (nothrow) int;

27

What is a shared_ptr?


Smart pointer.
Shared ownership.
Pointer deleted when last shared_ptr is destroyed.
Uses a reference count.

28

What is a unique_ptr?


Smart pointer.
Single ownership.
Object deleted when unique_ptr is destroyed.
Cannot be directly copied or assigned.

29

What is a weak_ptr?


Smart pointer.
Points to object managed by shared_ptr.
Not counted when shared_ptr deleted the object.
Must call lock in order to access the object.
e.g.

auto p = make_shared<int>(42);
weak_ptr<int> wp(p);
if (shared_ptr<int> np = wp.lock()) {
    // np shares object with p
}

30


When we use direct initialisation, we are asking the compiler to use ordinary function matching to select the constructor that best maches the arguments we provide.
When we use copy initialisation, we are asking the compiler to copy the right-hand operand into the object being created, converting that operand if necessary.

31


Copy initialisation ordinarily uses the copy constructor.
If a class has a move constructor, then copy initialisation sometimes uses that instead.

32

When does copy initialisation occur?


When defining a variable using an ‘=’.

  • Passing an object as an argument to a parameter of nonreference type.
  • Returning an object from a function that has a nonreference return type.
  • Brace initialising the elements in an array or the members of an aggregate class.

33


Library containers copy initialise their elements when you initialise the container or when calling insert of push.
Elements created by emplace are direct initialised.

34


A constructor that can be called with a single argument defines an implicit conversion from the constructor’s parameter type (x) to the class type (y).
That is, you can use an ‘x’ where an object of type ‘y’ is expected.

35

What rule of thumb can be used when deciding whether a class needs to define its own versions of the copy-control members?


Decide whether the class needs a destructor. Often more obvious.
If yes, it almost surely needs a copy constructor and copy-assignment operator as well.

36

How do you prevent copies of a class being made?


Define the copy constructor and copy-assignment operator as deleted functions.
e.g.

NoCopy(const NoCopy&) = delete;
NoCopy& operator=(const NoCopy&) = delete;

37

What is a deleted function?


One that is declared by may not be used in any other way.
Defined as deleted by following its parameter list with = delete.

38

What is a type-specific swap function?


Function written to swap the state of two objects of the same type.
Never necessary, but can be an important optimisation for classes that allocate.
Declare the swap function as a friend to give it access to the type’s private data members.
Because it exists to optimise our code, define swap as inline.
A type-specific version of swap will be a better match that the one in std.
If no type-specific version is defined then (assuming there is a using decl for swap in scope) calls to swap will use std::swap.
Each call to swap should be unqualified i.e. not std::swap.
e.g.

class MyType {
    friend void swap(MyType&, MyType&);
    //...
};
inline void swap(MyType &lhs, MyType &rhs) {
    using std::swap;
    swap(lhs.member, rhs.member);
    //...
}

39

What is ADL?


Argument-dependent lookup. aka Koenig lookup.
The set of rules for looking up the unqualified function names in a function-call expressions.
Makes it possible to use operators defined in a different namespace.
e.g.

#include <iostream>
//...
std::cout << "Test\n";      // OK: operator<< is found in std because the left arg is in std.
std::cout << endl;          // ERROR: not in global ns, not a func call to endl() so ADL does not apply.
end(std::cout);             // OK: func call, ADL examines std because of argument.
(endl)(std::cout);          // ERROR: 'end' not in global ns, sub-expression (endl) not a func call.

40

What is a type-specific swap function often used for?


To define the classes assignment operator.
The operator uses a technique known as ‘copy and swap’.
Swaps the left-hand operand with a copy of the right-hand operand.
e.g.

MyType& MyType::operator=(MyType rhs) {
    swap(*this, rhs);
    return *this;
} // 'rhs' is destroyed, which deletes what was previously in 'this'."
