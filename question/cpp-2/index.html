C++ Quiz #2

1

What is an lvalue?


An expression that yields an object or function.
A nonconst lvalue that denotes an object may be the left-hand operand of assignment.

2

What is an rvalue?


Expression that yields a value but not the associated location, if any, of that value

3

What is order of evaluation?


The order, if any, in which the operands to an operator are evaluated.
In most cases, compiler is free to evaluated operands in any order.
Only &&, ||, ?: and , specify the order.

4

What is precedence?


Order in which different operators in a compound expression are grouped.

5

What is short-circuit evaluation?


Used to describe how logical AND and OR operators execute.
If first operand is sufficient to determine result, evaluation stops.
Not guaranteed that second operand is NOT evaluated.

6

What is the , operator?


Binary.
Evaluated left-to-right.
Result is the value of the right-hand operand.
Result is lvalue if operand is lvalue.

7

What is the ^ operator?


Bitwise exclusive or.
Generates new integral value in which each position is 1 if either but not both operands contain a 1, otherwise 0.

8

What is the ~ operator?


Bitwise NOT operator.
Generates new integral value in which each bit is an inverted copy of those in the (possibly promoted) operand.

9

What is the << operator?


Left-shift operator.
Left-hand operand should be unsigned.
If the left-hand operand is signed, it is undefined if a shift causes a different bit to shift into the sign bit.

10

What is the >> operator?


Right-shift operator.
Opposite to left-shift operator.
If the left-hand operand is signed, it is implementation defined whether bits shifted into the result are 0 or a copy of the sign bit.

11

What is a dangling else?


Colloquial term.
The problem of how to process nested if statements in which there are more ifs than elses.
An else is always paired with the closest preceding unmatched if.
Use curly braces to control which if/else should match.

12

What is a null statement?


Indicated with a single semicolon.

13

What is are candidate functions?


Set of functions that are considered when resolving a function call.

14

What is a function prototype?


Function declaration.
A function prototype must have been declared before the point of calling it.

15

What is an initializer_list?


Library class that represents a comma-separated list of objects of a single type enclosed inside curly braces.

16

What is an abstract data type?


Data structure that encapsulates its implementation.

17

What is an access specifier?


Used to define whether members are accessible to users of the class.

18

What is an aggregate class?


Class with only public data members that has no in-class initialisation or constructors.
Members can be initialised by a brace-enclosed list of initialisers.
e.g.

struct Data {
    int ival;
    string s;
};
//...
Data val1 = { 0, "Rich" };

19

What is a const member function?


May not change an object’s ordinary data members.
The this pointer in a const member is a pointer to const.
const and nonconst functions may overload one another.

20

What is a converting constructor?


Nonexplicit constructor that can be called with a single argument.
Implicitly convert from the argument’s type to the class.

21

What is an explicit constructor?


Can be called with a single argument but cannot be used in an implicit conversion.
Made explicit by prepending the keyword explicit to its declaration.

22

What is a forward declaration?


Declaration of an as yet undefined name.
Often used to refer to a class that appears prior to its definition.

23

What is a friend?


Mechanism by which a class grants access to its nonpublic members.
Have same access rights as members.
Applies to both classes and functions.

24

What is an incomplete type?


Type that is declared, but not defined.
Not possible to use to define a variable or class member.
Is legal to use to define references or pointers.

25

What is a mutable data member?


Data member that is never const.
Can be changed inside a const function.
e.g.

class Screen {
public:
    void some_member() const;
private:
    mutable size_t access_ctr;
};
void Screen::some_member() const {
    ++access_ctr;
}

26

Where/when is = default used?


Used to signal to compiler that it should generate default constructor.
Used when class has other constructors.
Put after the parameter list of declaration of default constructor.

27

What is an fstream?


Read/write file stream.
Opened with in and out mode set by default.

28

What is an ifstream?


Reads a file stream.

29

What is an istringstream?


Reads a string stream.

30

What is an ofstream?


Writes to a file stream.

31

What is an ostringstream?


Writes to a string stream.

32

What is a stringstream?


Read/write string stream.

33

What is an adapter?


Library type, function or iterator that, given a type, function or iterator, makes it act like another.
Three sequential container adapters: stack, queue, priority_queue.

34

What is a sequential container?


Type that holds ordered collection of single type.
Elements are accessed by position.

35

What is a deque?


Sequential container.
Elements can be accessed by their positional index.
Support fast random access.
Like vector except supports fast insertion / deletion at the front and does not relocate elements due to insertions / deletions at either end.

36

What is a forward_list?


Sequential container.
Singly linked list.
Elements accessed only sequentially.
Iterators do not support ‘–’.
Supports fast insertion / deletion anywhere.
Unlike other containers, insertions / deletions can occur after a given iterator position.
As a result, it has a “before-the-beginning” iterator.
Iterators remain valid when new elements are added.
Iterators remain valid when elements are deleted, unless is iterator to deleted element.

37

What is a left-inclusive interval?


Range of values that includes its first element but not its last.
Typically denoted as [i, j), meaning starting at and including i, up to but excluding j.

38

What is a list?


Sequential container.
Doubly linked list.
Elements accessed only sequentially.
Iterators support both ‘–’ and ‘++’.
Supports fast insertion / deletion anywhere.
Iterators remain valid when new elements are added.
Iterators remain valid when elements are deleted, unless is iterator to deleted element.

39

What is an off-the-beginning iterator?


Iterator denoting the (nonexistent) element just before the beginning of a forward_list.
before_begin()
May not be dereferenced.

40

What is a priority_queue?


Adapter for sequential containers.
Elements not inserted at end, but based on specified priority level.
Default uses less-than operator for element type.

41

What is a queue?


Adapter for sequential containers.
Elements are added to back, removed from front.

42

What is a stack?


Adapter for sequential containers.
Elements are added / removed from one end only.

43

What is a vector?


Sequential container.
Elements accessed by their positional index.
Supports fast random access.
Elements can be added / removed efficiently at the back only.
Adding elements might cause it to be reallocated, invalidating all iterators.
Adding / removing elements in the middle invalidates all iterators to elements after that point.

44

What are iterator categories?


Iterators are classified depending on functionality they implement:-

  1. Input
  2. Output
  3. Forward
    • Extends Input, also extends Output (if nonconst).
  4. Bidirectional
    • Extends Forward, also supports backwards.
  5. Random Access
    • Extends Bidirectional, also access ranges non-sequentially.

45

What is a back_inserter?


Iterator adapter that takes a reference to a container and generates an insert iterator that uses push_back to add elements to the specified container.

46

What is a bidirectional iterator?


Same operations as forward iterators plus the ability to use ‘–’ to move backward.

47

What is the bind function?


Library function that binds one or more arguments to a callable expression.

#include <functional>

48

What is a capture list?


Portion of a lambda that specifies which variables from the surrounding context it can access.

49

What is the cref function?


Library function.
Returns a copyable object that holds a reference to a const object of a type that cannot be copied.

50

What is a front_inserter?


Iterator adapter.
Given a container, generates an insert iterator that uses push_front to add elements.

51

What are generic algorithms?


Type-independent algorithms.

52

What is an input iterator?


Can read, but not write elements of a sequence.

53

What is an insert iterator?


Iterator adapter.
Generates an iterator that uses a container operation to add elements to a given container.

54

What is an inserter?


Iterator adapter that takes an iterator and a reference to a container and generates an insert iterator that uses insert to add elements just ahead of the element referred to by the given iterator.

55

What is an istream_iterator?


Stream iterator that reads an input stream.

56

What are iterator categories?


Based on operations that an iterator supports.
Form a hierarchy, where more powerful iterators offer same operations as less categories.
Algorithms use categories to specify what operations the iterator arguments must support.
e.g. Some algorithms require only input iterators so won’t work on an iterator that only supports output iterator requirements.

57

What is a lambda expression?


Callable unit of code.
Like an unnamed, inline function.
Starts with a capture list, allowing access to variables in the enclosing function.
If body is a single return then type is inferred, otherwise defaults to void.

58

What is a move iterator?


Iterator adapter.
Generates an iterator that moves elements instead of copying them.

59

What is an ostream_iterator?


Iterator that writes to an output stream.

60

What is an output iterator?


Can write, but not necessarily read elements.

61

What is a predicate?


Function that returns a bool.
Often used by generic algorithms to test elements.
Those used by the library are either unary or binary.

62

What is a random-access iterator?


Same operations as bidirectional iterators plus relational operators to compare iterator values, and subscript operator and arithmetic operations on iterators.

63

What is the ref function?


Library function.
Generates a copyable object from a reference to an object of a type that cannot be copied.

64

What is a reverse iterator?


Iterator that moves backward through a sequence.
Exchanges the meaning of ++ and –.

65

66

What is a raw string and how do you define one?


String literal - used to avoid escaping of any character. Anything between the delimiters becomes part of the string.

R"(...........)"

67

Describe VC++ runtime library linking options and their differences?


  • /MT - Multi-threaded
  • /MTd - Multi-threaded Debug
  • /MD - Multi-threaded DLL
  • /MDd - Multi-threaded Debug DLL

68

With VC++, under what circumstances will an imported library not be linked to an executable? How might this be remedied?


When Visual C++ linker sees that nothing in the library is referenced from other object modules it throws the library out.
Export a function e.g. PullInMyLibrary, call it from the application or another library.

69

When must a class destructor be virtual?


If you intend to inherit from it - otherwise the destructor of the derived class will not be called when you delete an object through a base pointer.

70

When and how is the override keyword used?


Lets the compiler verify that you’ve properly overridden a function.
It follows the function parentheses.

71

Can a parameter containing a pointer return newly allocated memory to the caller?


No.
Parameters are passed by value, including the value held in a pointer.
If memory were to be allocated only the parameter would receive its address.
When the function returns the variable passed as a parameter would be unchanged.
The parameter must be made a pointer to a pointer.

72

What is a vtable?


Virtual function table.
C++ requires an additional lookup into the vtable (instead of simply calling a function).
In most cases the performance impact is negligible or even nonexistent. The compiler is able to optimise away some cost in certain cases.

73

What is often a more performant alternative to mocking via virtual functions?


Template-based. i.e.

template <typename DEPENDENCY>
class <class-under-test> {
private:
    DEPENDENCY dependency_;
};
typedef ClassUnderTest<ProductionType> ClassUnderTest;

74

How do you convert a reverse_iterator to an iterator?


iterator.base()

75

What does reverse_iterator::base() do?


Returns the underlying base iterator.
The base iterator refers to the element that is next to the element the reverse_iterator is currently pointing to.
i.e.

&*(rit.base() - 1) == &*rit

76

What is PIMPL?


“pointer to implementation”.
Extract concrete details to a separate implementation class.
Delegate to the implementation as needed from the interface functions.
You’re then free to change the implementation all you want, without triggering recompiles on the code dependent on the public interface.

e.g.

e.g.

ofstream file{filename, ios::out | ios::binary};
file.write(reinterpret_cast<char*>(&mydata), sizeof(MyData));

79

What is CppUTest’s leak detection? How does it work?


Compares memory at the start of its execution with memory at its completion and fails if there’s a mismatch.

80

What is linker substitution and when might it be used?


Create a library of stub functions for each one in use and link against that library.
In order to dummy-out a third-party library, esp. when writing tests.

81

What are the four keywords for type casting and in what scenarios would you use each of them?


  1. const_cast
  2. static_cast
  3. dynamic_cast
  4. reinterpret_cast

82

What does sizeof on an empty class return?


1 (byte).
The size of memory is decided by the compiler.

83

What does sizeof on a class with a constructor & destructor return?


1 (byte).
Function addresses are irrelevant to instances and compilers don’t add any instance data.

84

What does sizeof on a class with a constructor & virtual destructor return?


C++ compiler creates a virtual function table for the class and adds a pointer to the table in each instance.
4 bytes on a 32-bit system, 8 bytes on a 64-bit system - the size of a pointer.

85

What does sizeof on a class with a constructor & virtual destructor and a virtual method return?


C++ compiler creates a virtual function table for the class and adds a pointer to the table in each instance.
4 bytes on a 32-bit system, 8 bytes on a 64-bit system - the size of a pointer.

86

What is the result if you compile and execute the code below?

  1. Compiling error
  2. Compiles ok, but crashes.
  3. Compiles ok, executes with output of 10.
class A {
private:
    int value;
public:
    A(int n) {
        value = n;
    }
    A(A other) {
        value = other.value;
    }
    void Print() {
        std::cout << value << std::endl;
    }
};
int main(int argc, _TCHAR* argv[]) {
    A a = 10;
    A b = a;
    b.Print();
    return 0;
}

When the copy constructor is executed, it (recursively) calls the copy constructor due to the pass-by-value parameter.
Endless recursive calls cause a stack overflow.
Therefore, a pass-by-value parameter in a copy constructor is not allowed in the C++ standard.
Both Visual Studio & GCC report an error during compiling time. The copy constructor must be changed to:

A(const A& other)

87

Implement an assignment operator for the class below:

class CMyString {
public:
    CMyString(char* pData = NULL);
    CMyString(const CMyString& str);
    ~CMyString(void);
private:
    char* m_pData;
};


CMyString& CMyString::operator=(const CMyString& str) {
    if (this != &str) {
         CMyString strTemp(str);
         char* pTemp = strTemp.m_pData;
         strTemp.m_pData = m_pData;
         m_pData = pTemp;
    }
    return *this;
}

Key points to look out for:-

  • Does it return a reference? Without it, it’s not possible to chain assignments.
  • Is the argument passed by a constant reference? If it’s by value then a copy is made which is wasteful.
  • Is the existing memory freed?
  • Does it protect against self-assignment?
  • What if an exception is throw while reallocating memory? This would leave the object in an invalid state.

88

What are the differences between structs and classes?


The default access level in a struct is public, while in a class it is private.

89

Explain the following VC++ error:

“list initialization inside member initializer list or non-static data member initializer is not implemented”


The C++ compiler in Visual Studio does not implement list initialization inside either a member initializer list or a non-static data member initializer.
Visual Studio 2013 Update 3 reports this as an error.

90

What is a condition_variable?


Synchronization primitive that can be used to block one or more threads, until another thread notifies the condition variable.
Works only with std::unique_lock<std::mutex>.
Waiting threads must acquire std::unique_lock<std::mutex> on the same mutex used to protect the shared variable.
The wait operation atomically releases the mutex and suspends execution.
The working thread notifies the condition variable via notify_one() or notify_all().

91

What is the difference between a condition_variable and a condition_variable_any?


condition_variable works only with std::unique_lock<std::mutex>; allowing for maximal efficiency on some platforms.
condition_variable_any works with any BasicLockable object.

92

What is a unique_lock?


Mutex ownership wrapper.
It allows:

  • Deferred locking.
  • Time-constrained locking attempts.
  • Recursive locking.
  • Transfer of lock ownership.
  • Use with condition variables.

e.g.

#include <mutex>
std::mutex m;
std::unique_lock<std::mutex> lock(m);
// critical section...
lock.unlock();

93

What is a mutex? How is it used?


A lockable object, designed to signal when critical sections of code need exclusive access.
Prevents other threads with the same protection from executing concurrently.
Provide exclusive ownership.
Do not support recursivity.
e.g.

#include <mutex>
//...
std::mutex m;
lock(m);
// critical section.
m.unlock();

94

What is a standard-layout type?


A type with a simple linear data structure and access control that can easily be used to communicate with code written in other programming languages, such as C.
Defined with class, struct, or union.
Has no virtual functions or virtual base classes.
Has the same access control level for all its non-static data members.
Has no non-static data in the most derived class, max one base class with non-static data.
Its base class (if any) is a standard-layout class. And,
Has no base classes of the same type as its first non-static data member.

95

What is the deque template class?


Double-ended queue
Sequence container with dynamic size that can be expanded or contracted on both ends.
Allow for the individual elements to be accessed directly through random access iterators.

96

How does a deque differ from a vector?


Similar to vectors, but with efficient insertion and deletion at the beginning, not only at its end.
Vectors use a single array that needs to be reallocated occasionally for growth.
Elements of a deque can be scattered in different chunks of storage.
More complex internally than vectors.
Grow more efficiently under certain circumstances, especially with very long sequences, where reallocations become more expensive.

97

How do you define a type that may be used as though it were a function?


By defining member function operator() in the class.
e.g.

struct myclass {
    int operator()(int a) {return a;}
} myobject;
//...
int x = myobject(0);

98

When are function objects typically used?


As arguments to functions, such as predicates or comparison functions passed to standard algorithms.

99

What is the thread class?


An initialised thread object represents an active thread of execution.
An initialised thread object is joinable.
An initialised thread object has a unique thread id.
A default-constructed (non-initialised) thread object is not joinable and has a common thread id.
A joinable thread becomes not joinable if move-assigned, or join() or detach() is called.
Thread objects cannot be copied.
Calling the join() method on a thread causes the calling code to pause until the thread finishes.

100

What is a lock_guard used for?


Prevents concurrent access to code.
It’s a mutex wrapper that provides a convenient RAII-style mechanism for owning a mutex for the duration of a scoped block.
e.g.

#include <mutex>
void doItSafely() {
    std::lock_guard<std::mutex> block(mymutex);
    // from this point until the 'block' variable goes out of scope,
    // we're thread safe!
}
