The more hosts the hypervisor manages, the more resources it needs.

What is vertical scaling?


e.g. Getting a bigger box with faster CPU and better I/O can often improve latency and throughput, allowing you to process more work in less time.

3


For internal virtualization platforms, it is a common practice to have the virtual machine’s root partition mapped to a single SAN (storage area network). If that SAN goes down, it can take down all connected VMs.

4


it is important to know if a service-level agreement (SLA) is offered and plan accordingly. If you need to ensure your services are down for no more than four hours every quarter, but your hosting provider can only guarantee a downtime of eight hours per quarter, you have to either change the SLA, or come up with an alternative solution.

5


For its compute service, AWS offers only a 99.95% uptime over a given monthly period of the region as a whole, so you’ll want to distribute your workloads across multiple availability zones inside a single region.

6


because providers give you an SLA guarantee, they will tend to limit their liability! If them missing their targets costs you customers and a large amount of money, you might find yourself searching through contracts to see if you can claw anything back from them. Therefore, I would strongly suggest you understand the impact of a supplier failing in its obligations to you, and work out if you need to have a plan B (or C) in your pocket. More than one client I’ve worked with has had a disaster recovery hosting platform with a different supplier, for example, to ensure they weren’t too vulnerable to the mistakes of one company.

What is an ELB?


Elastic Load Balancer.
AWS provides them.

8


I have found myself advocating for software load balancers sitting behind the hardware load balancers to allow teams the freedom to reconfigure these as required. You do want to watch for the fact that all too often the hardware load balancers themselves are single points of failure

9

What ways can you ensure application data availability?


A common model would be using a standby. All data written to the primary database gets copied to the standby replica database.
If the primary goes down, my data is safe, but without a mechanism to either bring it back up or promote the replica to the primary, we don’t have an available database, even though our data is safe.

10


With proxy caching, a proxy is placed between the client and the server. A great example of this is using a reverse proxy or content delivery network (CDN).

11


With server-side caching, the server handles caching responsibility, perhaps making use of a system like Redis or Memcache, or even a simple in-memory cache.

12


Proxy caching is often a very simple way to add caching to an existing system. If the proxy is designed to cache generic traffic, it can also cache more than one service

13


In a situation where you have multiple types of clients, a server-side cache could be the fastest way to improve performance.

14


If we suffer a massive cache miss, perhaps because an entire machine (or group of machines) that provide our cache fail, a large number of requests will hit the origin.

15


For those services that serve up highly cachable data, it is common for the origin itself to be scaled to handle only a fraction of the total traffic, as most requests get served out of memory by the caches that sit in front of the origin. If we suddenly get a thundering herd due to an entire cache region vanishing, our origin could be pummelled out of existence.
One way to protect the origin in such a situation is never to allow requests to go to the origin in the first place. Instead, the origin itself populates the cache asynchronously when needed
If a cache miss is caused, this triggers an event that the origin can pick up on, alerting it that it needs to repopulate the cache.
We could decide to block the original request waiting for the region to be repopulated, but this could cause contention on the cache itself.
It’s more likely if we are prioritizing keeping the system stable that we would fail the original request, but it would fail fast.

16


the more caching you have, the harder it will be to assess the freshness of any piece of data. So if you think a cache is a good idea, keep it simple, stick to one, and think carefully before adding more

What is the syslog standard?


Standard for message logging.
Separation of the software that generates messages, the system that stores them, and the software that reports and analyzes them.

How do you disable the ‘Open File Security Warning’ when running an executable from a network share on Windows Server 2008 R2?


Run ‘Edit group policy’.
Navigate to “User Configuration\Administrative Templates\Windows Components\Attachment Manager”
Enable “Do not preserve zone information in file attachments”
Enable “Inclusion list for low file types”, adding “.exe” to the options field.

Which files does Mac OS X shell read during startup?


Reads and executes commands from the first that exists and is readable:-

  • ~/.bash_profile
  • ~/.bash_login
  • ~/.profile

20

What must you do if you have a .bash_profile and want a separate .profile?


Manually include it from ~/.bash_profile. Put the following in ~/.bash_profile:

source ~/.profile
